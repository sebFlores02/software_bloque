¿Qué diferencias y semejanzas hay entre Java y JavaScript? Java y JavaScript son dos lenguajes de programación diferentes. Java es un lenguaje compilado y orientado a objetos, utilizado para desarrollar aplicaciones de escritorio, móviles y web, mientras que JavaScript es un lenguaje interpretado y de scripting, principalmente utilizado en el desarrollo web para agregar interactividad y dinamismo. Java es de tipado estático y JavaScript de tipado dinámico. Ambos lenguajes comparten algunas similitudes en cuanto a su herencia común del lenguaje C, como variables, funciones y programación orientada a objetos.
¿Qué métodos tiene el objeto Date? El objeto Date se utiliza para trabajar con fechas y horas. Tiene multiples funciones como las son:

Get: Regresa de forma numerica el valor deseado

  • getDate()
  • getHours()
  • getMonth()

To: Cambia el formato de le fecha

  • toTimeString()
  • toLocaleDateString()
  • toLocaleTimeString()
¿Qué métodos tienen los arreglos? Los métodos más comunes que tienen los arreglos en JavaScript incluyen concat(), filter(), find(), forEach(), includes(), indexOf(), join(), map(), pop(), push(), reverse(), shift(), slice(), sort(), splice(), toString(), y unshift(). Estos métodos permiten realizar diversas operaciones en los arreglos, como agregar, eliminar o modificar elementos, ordenar los elementos, ejecutar funciones para cada elemento, y mucho más.
¿Cómo se declara una variable con alcance local dentro de una función? Para declarar una variable de forma local podemos hacer uso de cualquieras de las 3 "keyword" que tenemos disponible ya sea let, const o var. Estas tres variables no tienen el mismo alcance entre si, pero podemos usarlas para limitar el alcance de cualquier variable a una funcion. El metodo de var es mas antiguo, const es para definir una variabale cuyo su valor no podra ser modificado despues de su declaracion y let tiene un alcance distinto donde tambien se limita a un bloque.
¿Qué implicaciones tiene utilizar variables globales dentro de funciones? Utilizar variables globales dentro de funciones en JavaScript puede tener implicaciones negativas en el código, ya que puede generar confusión, colisiones de nombres, dificultades para rastrear errores y falta de modularidad. Se recomienda utilizar variables locales en lugar de variables globales siempre que sea posible, ya que son más precisas en cuanto a su alcance y uso dentro de una función, lo que hace que el código sea más fácil de entender, depurar y mantener.td>
